Client Report

Client: McMaster Computer Science Society (CSS)
Overview

CSS is a student-led academic society at McMaster University, where they strive to organize events to help the CS community network among themselves. However, they faced difficulties monitoring progress and coordinating events, as they mainly use chat boxes; however, this is insufficient, as it causes members to scroll up and regularly monitor chat logs to know event timings. In addition, they have biweekly meetings to check in with members and monitor productivity; however, this is all inadequate since there is no way to monitor past work, as there is no record of it. Hence, we (Built by Four) have decided to create an event management & dashboard website that allows many other organizations like the CSS to monitor productivity and manage events, which improves efficiency and sparks growth.

App Details
Target Users

The target users are mainly organizations/clubs as they would use our application to increase efficiency and monitor productivity of the members. They are most likely desktop users, however condensing employee stats and event schedule is not ideal for mobile users. They may also need easy-to-read dashboards so both managers and members can use the platform effectively.

Static/Basic Information
  • Organization Identity
    • Computer Science Society name
    • Short description / mission statement
    • McMaster University affiliation
  • Contact & Communication
    • Official CS Society email
    • Discord/Instagram/LinkedIn links
  • Executive Team Information
    • Executive names
    • Roles (President, VP Events, etc.)
    • Term (e.g., 2024–2025)
  • Event Details
    • For each event (title, date & time, location, short description)
Design Elements

The client prefers a clean and modern design with minimalistic visuals to reduce confusion to accommodate ones who are not used to technology. The client does not want to overcomplicate things and add a lot of features and buttons as it will increase the time to learn how to use our product. In terms of the event they would like a simple card-based layout for events and a clear hierarchy (title, date, location, etc).

Data for Client

The main information the app will manage is event information which includes the event name (date, time, location, and description), event capacity, and RSVP/interest count to determine the number of students interested. However, there is additional information that would be effective to have such as anonymous RSVP details, event categories, and statistics to determine how many events are hosted. Team stats regarding how the event was marketed would be important to have since there are many sub teams that put work into publishing the event. For example, social media, web design, and outreach.

Interactive Elements

The most important interactive element the client would like is an RSVP/interest button counter to show how many people are interested in attending, which would change to "interested" when a certain number of people have expressed interest in the event. Additionally, a filtering drop-down menu is available to help participants select the event they prefer based on category. For the dashboard & productivity, having interactive counters and bar graphs to show the number of events hosted, average RSVP count, and attendance trends across a certain period. To add on, having visual progress indicators, for example, events planned vs completed, monthly event count, and how many individuals worked on the event, are required for the executive productivity indicator.
